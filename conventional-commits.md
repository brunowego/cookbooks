## Pattern

### Usage

```text
type(category): description [flags]
```

#### Allowed `<type>` values

- breaking
- build
- ci
- chore
- docs
- feat
- fix
- other
- perf
- refactor
- revert
- style
- test

#### Example `<scope>` values

- init
- runner
- watcher
- config
- web-server
- proxy
- etc.
